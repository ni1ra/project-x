## 1. The pre-registered predicate

The test:

> The Project X agent (entrypoint `ReasoningAgent.process(prompt)` from `src/project_x/reasoning_agent.py`) is run against 100 held-out math worked-example prompts auto-generated by a deterministic parameter grid (seed=42). Each prompt is independently graded with one of four labels:
>
> - **CORRECT** — the agent's final answer matches the mechanically-computed ground-truth within numerical tolerance (`abs(actual - expected) < 1e-6`).
> - **REFUSED** — the agent emits `confidence='refused'` OR returns no final-answer numeric. This is honest failure preferred over confabulation per M-PROJECTX-013.
> - **CONFIDENTLY-WRONG** — the agent emits `confidence='high'` (or non-refused) AND the final numeric mismatches ground truth by more than tolerance.
> - **PARTIALLY-CORRECT** — agent's chain shows correct intermediate steps but final numeric is off (e.g., sign error on otherwise correct work). Distinct from CONFIDENTLY-WRONG.
>
> The verdict on Project X math capability at the run-time of this predicate is determined by the percentage of CORRECT prompts:
>
> - **STRONG** if **≥ 80% CORRECT** out of 100 — the substrate (whether hand-coded scaffolding or learned-from-corpus model) reliably solves the test set.
> - **PARTIAL** if **30-80% CORRECT** out of 100 — the substrate solves some prompts but has gaps.
> - **NOT VALIDATED** if **< 30% CORRECT** out of 100 — substrate cannot reliably solve the test set even after whatever cycle-N+M work landed.

Three-band verdict refuses the binary trap. Distinct labels (REFUSED vs CONFIDENTLY-WRONG vs PARTIALLY-CORRECT) capture failure modes that uniform "correct / not correct" obscures — particularly important for the M-PROJECTX-013 measure-don't-claim discipline (CONFIDENTLY-WRONG is the failure shape that auto-passes a binary correct-or-not test).

Each generator method produces N prompts; total 100. The substrate generator function `generate_math_predicate_test_set(seed=42)` (to be implemented at cycle-15 implementation phase if B3 picks) returns the 100 prompts + per-prompt ground-truth values.

## 3. Methodology (pre-registered)

1. Generate the 100 prompts via `generate_math_predicate_test_set(seed=42)`.
2. For each prompt:
   - Capture `agent.process(prompt).answer_text`.
   - Extract final numeric answer from the answer text (per substrate-specific extraction: `solve_quadratic` outputs `[r1, r2]`; `free_fall_time` outputs a single float; etc.).
   - Compare against ground truth: `abs(extracted - expected) < 1e-6` → CORRECT; else inspect agent_response.confidence + intermediate state → REFUSED / PARTIALLY-CORRECT / CONFIDENTLY-WRONG.
3. Aggregate counts: N_correct / N_partial / N_refused / N_confidently_wrong.
4. Compute `correct_pct = N_correct / 100`.
5. Apply verdict bands from §1: STRONG / PARTIAL / NOT VALIDATED.
6. Report run results in a separate **§7 Result block** appended to this doc at run-time (NOT retro-edited into §§1-6).

## 4. Why this predicate is honest

- **Pre-registered before data exists.** Verdict isn't a post-hoc story.
- **Falsifiable.** Three labelled outcome bands; CONFIDENTLY-WRONG distinct from REFUSED captures the M-PROJECTX-013 failure shape.
- **Threshold-explicit.** 80% / 30% / <30% — numbers, not "many / most / some."
- **Mechanical-ground-truth.** Substrate generator + auto-graded — no rubric ambiguity (M-PROJECTX-014 mechanical-domain side).
- **Test set pre-published.** The parameter grid is in §2 BEFORE the test set is generated. The author cannot retro-pick parameters to favor a desired verdict.
- **Anticipated failure mode named:** the cycle-15 #01 demo (commit `c197bbd`) F1 finding is that the formal `pendulum_period` parser was BYPASSED on a prose-form pendulum prompt. If similar parser-prose-form mismatches plague the quadratic / matrix / integral generators, the predicate will surface them as REFUSED count (parser-miss) or CONFIDENTLY-WRONG count (natural-mode-walk over math corpus instead of formal route). The predicate's distinct labels capture this.
- **Three-band verdict.** Strong / partial / none — refuses the binary trap.

## 5. Anticipated failure modes (pre-registered before run)

The cycle-15 #01 demo (commit `c197bbd`) already surfaced two findings the math predicate will likely encounter:

- **F1-style: formal-route bypass on prose-form prompt.** If the parameter-grid generator produces prompts the formal parser DOESN'T recognize (e.g., the quadratic generator emits `"Find the roots of 1.0x² + -7.0x + 12.0 = 0"` — sign normalization issues), the BG-dispatcher's combined-confidence ranking sends the prompt to natural-mode walk. If the walk produces the right answer by accident → CORRECT; if the walk produces narrative-prose corpus material → CONFIDENTLY-WRONG (the walk text is "high-confidence" by the substrate's measure but mismatches the ground truth).
- **F3-style: `combined_confidence=1.0` saturation via missing-archetype default.** Same mechanism the cycle-14 #08e deferred-finding flagged. If ANY natural-mode walk domain has no archetype entry, the dispatcher saturates and walks confidently in the wrong domain. The predicate will flag this as CONFIDENTLY-WRONG even if the agent's `confidence` field is `provenance-traced` rather than `high` — the predicate's CONFIDENTLY-WRONG label captures "any non-refused emission with wrong final answer," not just `confidence='high'`.

These failure modes are pre-registered ANTICIPATED, not retro-explained — the cycle-15 #01 demo evidence is on disk + cited in the predicate spec BEFORE any predicate run executes.
