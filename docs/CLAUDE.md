# docs/

## What lives here

Project documentation root — universal layout per `~/.claude/CLAUDE.md` § UNIVERSAL VS PROJECT-SPECIFIC.

## Files

| File | Purpose |
|---|---|
| `A_TO_Z_PLAN.md` | Live phase plan (current). Rotates to `past_work/phases/` on phase exit. |
| `DO_THIS_NEXT.md` | Cycle handoff brief. Rewritten per cycle close. |
| `MANIFESTO.md` | lain's intent for Project X — heartbeat-tracked, dynamic doc. New global standing-order 2026-05-10 03:10. |
| `artifacts/` | Phase verdicts + frozen research notes (PHASE_*_*.md). |
| `past_work/phases/` | Archived phase plans (one per completed phase: phase_1 through phase_11). |
| `past_work/cycles/phase_<N>/` | Per-cycle reflections (`dev-cycle-<M>.md` per cycle close). |

## Conventions

- `CLAUDE.md` is the only `.md` allowed in repo root (per `~/.claude/CLAUDE.md` standing layout).
- Per-cycle reflections write directly to `past_work/cycles/phase_<N>/dev-cycle-<M>.md` from cycle 1 of a phase — no later move/consolidate (lain 2026-05-02; phase-named files in cycles/ were confusing, looked like phase plans).
- Phase exit archives `A_TO_Z_PLAN.md` → `past_work/phases/phase_<N>_<theme>.md`. Then fresh `A_TO_Z_PLAN.md` is written for the next phase.
- `MANIFESTO.md` is dynamic — heartbeat reconciles freshness; cycle 8 verdict references it.
